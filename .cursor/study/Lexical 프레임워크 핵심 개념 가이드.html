Lexical 프레임워크 스터디 가이드

퀴즈 (단답형 질문)

각 질문에 대해 2-3문장으로 간결하게 답하세요.

  1. Lexical에서 $function 컨벤션이 필요한 이유는 무엇이며, ESLint 플러그인이 이 컨벤션을 어떻게 시행하나요?
  2. Lexical에서 EditorState를 "소스 오브 트루스"로 간주하는 주된 이유는 무엇이며, HTML과 비교했을 때 어떤 이점이 있나요?
  3. @lexical/headless 패키지는 어떤 용도로 사용되며, 일반 Lexical 에디터와 어떤 주요 차이점이 있나요?
  4. editor.update() 와 editor.registerNodeTransform() 의 주된 차이점은 무엇이며, 성능 측면에서 어떤 것을 선호해야 하나요?
  5. @lexical/history 패키지에서 registerHistory 함수는 어떤 역할을 하며, 어떤 명령어를 처리하나요?
  6. @lexical/html 패키지에서 Lexical 노드를 HTML로 내보낼 때 exportDOM() 메서드의 역할은 무엇인가요?
  7. Lexical의 NodeState API는 기존 노드 속성 정의 방식과 비교하여 어떤 이점을 제공하나요?
  8. Lexical에서 RootNode 는 다른 ElementNode 서브클래스와 어떻게 다른 특성을 가지나요?
  9. 협업 환경에서 Yjs 모델을 "소스 오브 트루스"로 사용하는 것이 권장되는 이유는 무엇인가요?
  10. Lexical에서 '연속적인 타이핑'에 대한 되돌리기(undo) 병합(coalescing)은 무엇을 의미하며, 어떤 요소들이 연속성을 끊을 수 있나요?

퀴즈 정답

  1. Lexical에서 $function 컨벤션은 특정 컨텍스트(예: editor.update() 클로저 내부)에서만 호출될 수 있는 함수를 명확하게 표시하기 위해 사용됩니다. @lexical/eslint-plugin 은 이 컨벤션을 강제하여 개발자가 올바른 컨텍스트에서 이러한 함수를 사용하도록 돕고 런타임 오류를 방지합니다.
  2. Lexical에서 EditorState는 DOM이 아닌 자체적인 내부 상태 모델을 소스 오브 트루스로 유지합니다. 이는 HTML이 텍스트 편집에 너무 유연하여 다양한 표기법이 동일한 결과물을 내는 비정규화 문제를 해결하고, 콘텐츠 구조와 포맷팅을 분리하여 정규화된 문서 구조를 유지하는 데 이점이 있습니다.
  3. @lexical/headless 패키지는 DOM에 의존하지 않는 환경(예: Node.js)에서 Lexical 에디터와 상호 작용할 수 있도록 해줍니다. 이를 통해 서버 측에서 에디터 상태를 생성, 업데이트, 순회하거나 마크다운 변환과 같은 작업을 수행할 수 있습니다.
  4. editor.update() 는 에디터 상태를 변형하기 위해 함수를 전달받는 주요 방법이지만, DOM 조정을 비동기적으로 스케줄링하여 성능을 최적화합니다. 반면 editor.registerNodeTransform() 는 노드 변경에 반응하여 동일한 업데이트의 일부로 변형을 수행하므로 추가적인 렌더링을 유발하지 않아 성능상 더 효율적입니다.
  5. @lexical/history 패키지의 registerHistory 함수는 실행 취소/다시 실행(undo/redo) 기록 스택과 관련 에디터 명령어를 관리하는 데 필요한 리스너를 등록합니다. 이 패키지는 UNDO_COMMAND , REDO_COMMAND , CLEAR_HISTORY_COMMAND 를 처리하며, 기록 상태가 변경될 때 CAN_UNDO_COMMAND 와 CAN_REDO_COMMAND 명령어를 트리거합니다.
  6. exportDOM() 메서드는 LexicalNode 가 HTML로 어떻게 표현되어야 하는지를 제어합니다. 이 메서드는 에디터 상태를 HTML로 변환할 때 각 노드를 HTMLElement 로 변환하기 위해 호출되며, 변환 후 추가적인 처리를 위한 'after' API도 제공합니다.
  7. NodeState API는 Lexical v0.26.0에 도입되어 어떤 노드에든 임의의 상태를 즉석에서 추가할 수 있도록 합니다. 이는 JSON 직렬화, 기록(history), 조정(reconciliation)에 자동으로 참여하며, 기존 방식보다 보일러플레이트 코드를 줄이고 루트 노드에 문서 수준 메타데이터를 저장하는 등 유연성을 높입니다.
  8. RootNode 는 EditorState 에 단 하나만 존재하며 항상 최상위에 있고, 부모나 형제 노드를 가지지 않습니다. ElementNode 의 서브클래스와 달리 서브클래싱되거나 교체될 수 없으며, 변형 리스너가 아닌 루트 레벨 또는 업데이트 리스너를 통해 문서 수준의 변경 사항을 관찰해야 합니다.
  9. Yjs 모델을 소스 오브 트루스로 사용하는 것이 권장되는 이유는 클라이언트가 인터넷 연결 없이도 서버와 안정적으로 동기화하고 참여할 수 있는 유일한 방법이기 때문입니다. 또한, 데이터베이스와 달리 Yjs 모델 자체에 협업에 필요한 모든 정보가 포함되어 있어 상태 관리가 용이합니다.
  10. '연속적인 타이핑'에 대한 되돌리기 병합은 사용자가 연속적으로 문자를 입력하거나 지우는 등의 동작을 하나의 되돌리기 세션으로 묶어, 한 번의 되돌리기 동작으로 해당 연속적인 편집을 모두 취소할 수 있도록 하는 기능입니다. 시간 초과, 사용자 선택 변경, 키보드 포커스 변경, 그리고 복사 및 붙여넣기와 같은 동작은 연속성을 끊고 새로운 되돌리기 세션을 시작하게 합니다.

에세이 형식 질문

  1. Lexical의 핵심 디자인 원칙인 "소스 오브 트루스는 DOM이 아닌 내부 상태 모델"이라는 개념이 에디터의 신뢰성, 접근성, 성능에 어떻게 기여하는지 설명하고, 이로 인해 얻을 수 있는 개발자 경험의 이점을 논하세요.
  2. Lexical에서 플러그인을 생성하는 두 가지 주요 방법(프레임워크 독립적인 플러그인과 React 플러그인)을 비교하고, 각각의 장단점 및 적절한 사용 시나리오에 대해 설명하세요. 특히, LexicalComposer 와 useLexicalComposerContext 의 역할에 초점을 맞추세요.
  3. Lexical의 명령(Command) 시스템이 에디터의 확장성과 유연성에 어떻게 기여하는지 설명하고, createCommand() , dispatchCommand() , registerCommand() 함수가 상호 작용하여 에디터 기능을 구현하는 과정을 구체적인 예시와 함께 서술하세요.
  4. Lexical의 노드(Node) 시스템에서 Key Management 의 중요성을 설명하고, 노드의 __key 속성이 내부적으로 어떻게 사용되는지, 그리고 개발자가 __key 를 직접 조작하지 않아야 하는 이유와 올바른 노드 복제( $copyNode ) 및 수정( getWritable() ) 방법을 설명하세요.
  5. Lexical의 Node Transforms 는 에디터 상태 변경에 반응하는 가장 효율적인 메커니즘으로 설명됩니다. Node Transforms 의 Transform heuristic 과 Preconditions 개념을 상세히 설명하고, update listener 를 사용하는 것이 왜 권장되지 않는지 성능상의 이유와 함께 설명하세요.

용어 사전

  • @lexical/clipboard : Lexical의 클립보드 기능(복사 및 붙여넣기)을 포함하는 패키지입니다.
  • @lexical/code : Lexical의 코드 블록 및 코드 하이라이팅 기능을 포함하는 패키지입니다.
  • @lexical/code-shiki : shikijs 엔진을 사용하여 Lexical의 코드 하이라이터를 제공하는 패키지입니다.
  • @lexical/devtools-core : Lexical 디버깅 및 개발에 필요한 도구를 포함하는 패키지입니다.
  • @lexical/dragon : Dragon NaturallySpeaking 접근성 도구와의 호환성을 제공하는 패키지입니다.
  • @lexical/eslint-plugin : Lexical의 $function 컨벤션을 강제하는 ESLint 플러그인입니다.
  • @lexical/file : Lexical의 파일 가져오기/내보내기 기능을 포함하는 패키지입니다.
  • @lexical/hashtag : Lexical 해시태그 기능을 포함하는 패키지입니다.
  • @lexical/headless : DOM에 의존하지 않는(예: Node.js) 환경에서 Lexical과 상호 작용할 수 있도록 하는 패키지입니다.
  • @lexical/history : Lexical의 되돌리기/다시 실행 기록 스택을 관리하는 기능을 포함하는 패키지입니다.
  • @lexical/html : Lexical <-> HTML 변환 유틸리티 함수를 내보내는 패키지입니다.
  • @lexical/link : Lexical 링크 기능을 포함하는 패키지입니다.
  • @lexical/list : Lexical에서 목록(리스트)을 구현하기 위한 기본 기능을 제공하는 패키지입니다.
  • @lexical/mark : Lexical에서 콘텐츠를 마크로 감싸는 데 사용되는 헬퍼 및 노드를 포함하는 패키지입니다.
  • @lexical/markdown : Lexical의 마크다운 가져오기, 내보내기, 단축키 기능을 포함하는 패키지입니다.
  • @lexical/offset : Lexical의 선택 오프셋 헬퍼를 포함하는 패키지입니다.
  • @lexical/overflow : Lexical의 선택 오버플로 헬퍼 및 노드를 포함하는 패키지입니다.
  • @lexical/plain-text : 기본적인 텍스트 편집 동작(입력, 삭제, 복사/붙여넣기)을 위한 리스너를 등록하여 Lexical 사용을 시작할 수 있는 패키지입니다.
  • @lexical/react : React 애플리케이션에서 텍스트 편집을 위한 Lexical 컴포넌트 및 훅 세트를 제공하는 패키지입니다.
  • @lexical/rich-text : 기본적인 텍스트 편집 동작 외에 제목, 서식 있는 텍스트, 인용구 등 리치 텍스트 기능을 위한 기본 동작을 제공하는 패키지입니다.
  • @lexical/selection : Lexical의 선택 헬퍼를 포함하는 패키지입니다.
  • @lexical/table : Lexical의 테이블 기능을 포함하는 패키지입니다.
  • @lexical/text : Lexical 텍스트 처리를 위한 유틸리티 및 헬퍼를 포함하는 패키지입니다.
  • @lexical/utils : Lexical의 기타 유틸리티를 포함하는 패키지입니다.
  • @lexical/yjs : Lexical을 사용한 협업 편집을 위한 Y.js 바인딩 세트를 제공하는 패키지입니다.
  • $function : Lexical에서 특정 컨텍스트(예: editor.update() 또는 editor.read() 클로저 내부)에서만 호출될 수 있는 함수들을 가리키는 컨벤션입니다. 함수 이름 앞에 달러 기호($)가 붙습니다.
  • $getSelection() : 현재 EditorState의 선택(Selection) 객체를 가져오는 $function 입니다.
  • $getRoot() : EditorState의 루트 노드(RootNode)를 가져오는 $function 입니다.
  • $copyNode() : 기존 노드의 새 복사본을 새 키로 생성하는 공개 API입니다.
  • $generateHtmlFromNodes() : Lexical 에디터의 노드로부터 HTML 문자열을 생성하는 함수입니다.
  • $generateNodesFromDOM() : DOM 인스턴스에서 Lexical 노드를 생성하는 함수입니다.
  • $getState() : 특정 노드에서 NodeState 값을 가져오는 함수입니다.
  • $setState() : 특정 노드에 NodeState 값을 설정하는 함수입니다.
  • __key : Lexical 에디터 내 각 노드에 할당된 고유 식별자로, 노드 추적 및 효율적인 상태 관리에 사용됩니다.
  • $applyNodeReplacement : 새로운 노드를 생성하고 노드 교체 설정을 적용하는 데 사용되는 유틸리티 함수입니다.
  • editor.dispatchCommand() : Lexical 명령을 실행하는 메서드입니다.
  • editor.getEditorState() : 현재 에디터의 최신 EditorState 객체를 반환하는 메서드입니다.
  • editor.isEditable() : 에디터의 현재 편집 모드(읽기/편집)를 반환하는 메서드입니다.
  • editor.parseEditorState() : 문자열화된 EditorState JSON을 파싱하여 EditorState 객체를 생성하는 메서드입니다.
  • editor.registerCommand() : 특정 Lexical 명령에 대한 리스너를 등록하는 메서드입니다.
  • editor.registerDecoratorListener() : 에디터의 데코레이터 객체 변경 시 알림을 받는 리스너입니다.
  • editor.registerEditableListener() : 에디터의 편집 모드 변경 시 알림을 받는 리스너입니다.
  • editor.registerMutationListener() : 특정 유형의 Lexical 노드가 변형(생성, 파괴, 업데이트)될 때 알림을 받는 리스너입니다.
  • editor.registerNodeTransform() : 노드 유형의 변경에 반응하여 에디터 상태를 효율적으로 변형하는 리스너를 등록합니다.
  • editor.registerRootListener() : 에디터의 루트 DOM 요소 변경 시 알림을 받는 리스너입니다.
  • editor.registerTextContentListener() : 에디터의 텍스트 콘텐츠가 변경될 때 알림을 받는 리스너입니다.
  • editor.registerUpdateListener() : Lexical이 DOM에 업데이트를 커밋할 때 알림을 받는 리스너입니다.
  • editor.setEditable() : 에디터의 편집 모드를 설정하는 메서드입니다.
  • editor.setEditorState() : 현재 에디터 상태를 인수로 전달된 상태로 대체하는 메서드입니다.
  • editor.setRootElement() : 에디터 인스턴스를 contenteditable DOM 요소에 연결하는 메서드입니다.
  • editor.update() : 에디터 상태를 변형하기 위한 가장 일반적인 방법으로, 클로저 내에서 $function 을 사용하여 작업을 수행합니다.
  • exportJSON() : LexicalNode가 JSON으로 어떻게 표현되어야 하는지 제어하는 메서드입니다.
  • getWritable() : 노드의 변경 가능한 버전을 가져오는 내부 API입니다. 노드 메서드 내에서 사용됩니다.
  • importDOM() : HTMLElement 가 Lexical에서 어떻게 표현되어야 하는지 제어하는 LexicalNode 의 정적 메서드입니다.
  • importJSON() : JSON으로부터 LexicalNode가 어떻게 역직렬화되어 노드로 변환되는지 제어하는 메서드입니다.
  • initialConfig : Lexical 에디터를 초기화할 때 사용되는 구성 객체로, 네임스페이스, 테마, 노드 등록, 오류 처리 등을 정의합니다.
  • LexicalCommand : Lexical의 명령 시스템에서 사용되는 명령 유형으로, 특정 액션을 나타냅니다.
  • LexicalComposer : React 애플리케이션에서 Lexical 에디터를 설정하기 위한 최상위 컴포넌트입니다. 에디터 인스턴스를 컨텍스트로 제공합니다.
  • LexicalEditor : 모든 Lexical 기능의 핵심 인스턴스입니다. contenteditable DOM 요소에 연결됩니다.
  • LexicalErrorBoundary : React 컴포넌트에서 Lexical 업데이트 중 발생하는 오류를 처리하기 위한 오류 경계 컴포넌트입니다.
  • LexicalNode : Lexical의 핵심 기본 노드로, 에디터의 시각적 뷰와 기본 데이터 모델을 형성합니다.
  • LexicalPlainTextPlugin : @lexical/plain-text 의 React 래퍼로, 일반 텍스트 편집 기능을 제공합니다.
  • LexicalRichTextPlugin : @lexical/rich-text 의 React 래퍼로, 리치 텍스트 편집 기능을 제공합니다.
  • ListNode : Lexical에서 목록을 나타내는 노드입니다.
  • ListItemNode : Lexical에서 목록 항목을 나타내는 노드입니다.
  • NodeCaret : Lexical 문서 트리를 효율적으로 탐색하기 위한 통일된 API입니다. 문서 내 특정 지점을 origin , direction , type 으로 표현합니다.
  • NodeReplacement : 에디터 내에서 주어진 노드의 모든 인스턴스를 서브클래스의 인스턴스로 교체할 수 있도록 하는 기능입니다.
  • NodeSelection : 여러 개의 임의의 노드를 선택하는 것을 나타내는 선택 유형입니다.
  • NodeState : Lexical v0.26.0에 도입된 API로, 임의의 상태를 노드에 즉석에서 추가할 수 있으며, 이는 조정(reconciliation), 기록(history), JSON 직렬화에 참여합니다.
  • PlainTextPlugin : @lexical/plain-text 를 감싸는 React 컴포넌트입니다.
  • RangeSelection : 브라우저의 DOM Selection 및 Range API를 정규화한 가장 일반적인 선택 유형입니다. anchor , focus , format 속성을 가집니다.
  • reconciliation : Lexical이 EditorState 간의 "차이(diff)"를 적용하여 DOM의 변경이 필요한 부분만 업데이트하는 과정입니다.
  • RootNode : EditorState에서 항상 최상위에 존재하는 단일 노드로, contenteditable 요소 자체를 나타냅니다. 서브클래싱되거나 교체될 수 없습니다.
  • TableSelection : 테이블과 같은 그리드 형태의 선택을 나타내는 선택 유형입니다.
  • TextNode : 텍스트를 포함하는 리프(leaf) 유형의 노드입니다. format , mode , style 등의 텍스트 관련 속성을 가집니다.
  • Transformer : 마크다운 가져오기, 내보내기, 또는 타이핑 중 특정 텍스트나 노드가 처리되는 방식을 정의하는 객체입니다.
  • updateFromJSON() : importJSON 구현을 단순화하기 위해 Lexical 0.23에 도입된 메서드로, JSON을 기반으로 노드의 속성을 설정합니다.
  • Update Tags : 업데이트에 첨부할 수 있는 문자열 식별자로, 업데이트의 유형이나 목적을 나타내고 리스너에 의해 처리 방식을 제어하는 데 사용됩니다.
  • useLexicalComposerContext() : React Context를 통해 LexicalEditor 인스턴스에 접근할 수 있도록 하는 React 훅입니다.
  • Y.Doc : Y.js의 핵심 데이터 구조로, 여러 클라이언트 간에 상태를 공유하고 동기화하는 데 사용됩니다.
  • Yjs : 실시간 협업 편집을 위한 프레임워크입니다.
